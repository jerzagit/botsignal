Guard 6 · Lot Calculation
—
Calculated per signal
details
Blocks if calculated lot is 0 — account too small for even MIN_LOT. Lot = (free_margin × risk%) / (SL_pips × pip_value). Weighted DCA split: L1 gets 30%, remaining 70% shared across L2+.
PASS: $500 margin, 10% risk, 50p SL → 0.09 lot L1 (30%): 0.03 lot → 3 sub-orders × 0.01 L2 (35%): 0.03 lot → 3 sub-orders × 0.01 L3 (35%): 0.03 lot → skipped (runway guard) PASS: $1000, 0.20 lot → L1=0.06, L2=0.07, L3=0.07 BLOCK: $50 margin → 0.005 lot < MIN_LOT → blocked

Profit Lock
—
Monitors open positions
details
When any position reaches +50 pips profit, SL moves to breakeven (entry price) and TP tightens to entry ± 100p. Checked every 30s. After locking, trailing stop activates automatically.
SELL entry @ 4651, profit lock at +50p (price = 4646): SL → 4651 (breakeven, zero risk) TP → 4641 (entry − 100p) → Trailing stop now active (20p trail) Price drops to 4635 (+20p further): Trail kicks in → SL moves to 4637 → Profit guaranteed even if price reverses
Trailing Stop
—
Activates after profit lock
details
After profit lock fires, SL trails price every 20 pips of further favorable movement. SL only moves in profit direction — never backwards. Locks in progressively more profit as price runs.
SELL locked @ 4651 (SL=BE), price now 4646: trail_price seeded = 4646 Price → 4644 (2p move, < 20p threshold → no trail) Price → 4628 (18p move below seed → trail!) New SL = 4628 + 20p = 4630 trail_price updated = 4628 Price → 4610 (18p below 4628 → trail!) New SL = 4610 + 20p = 4612 → Minimum profit locked: entry−4612 = 39p
DCA Sub-Splitting
—
Calculated per signal
details
Weighted lot split: L1 gets 30% of total lot, remaining 70% shared equally across L2+. Each layer sub-splits into orders with TPs distributed in thirds: TP1 (quick), TP2 (medium), Runner (no TP).
Weighted split (L1_LOT_RATIO = 30%): Total 0.09 lot, 3 layers: L1 = 0.09 × 30% = 0.03 → 3 sub-orders × 0.01 L2 = 0.09 × 35% = 0.03 → 3 sub-orders × 0.01 L3 = 0.09 × 35% = 0.03 → skipped (runway) TP thirds per layer: 1 split: 1×TP1 2 splits: 1×TP1, 1×TP2 3 splits: 1×TP1, 1×TP2, 1×Runner 4 splits: 1×TP1, 2×TP2, 1×Runner Runner = no TP, rides with trailing stop + BE
DCA Gap & Runway
—
Applied per DCA session
details
Layer gap scales with SL size instead of using a fixed pip distance. Runway guard skips any layer whose trigger is too close to SL — prevents entries with no room to breathe.
Dynamic gap: gap = SL × ratio 50p SL × 0.40 = 20p gap between layers 40p SL × 0.40 = 16p gap Runway guard: skip if trigger < min runway from SL L3 trigger = entry - 40p, SL = entry - 50p Runway = 10p < 25p minimum → L3 SKIPPED If L2_GAP_RATIO = 0 → falls back to fixed LAYER2_PIPS

Fib Retracement Guard
—
Checked on manual trades
details
Checks if price is in the 0–38.2% Fibonacci retracement zone of the last opposite H1 candle. Ensures you're buying dips and selling rallies — not chasing.
BUY check: Find last bearish H1 candle (close < open) 0% = candle low (best buy), 100% = candle high Price at 15% → ✅ in zone (buying the dip) Price at 60% → ❌ above 38.2% (chasing) SELL check: Find last bullish H1 candle (close > open) 0% = candle high (best sell), 100% = candle low Price at 10% → ✅ in zone (selling the rally) Outside zone → CONFIRM/CANCEL warning Combined with trend warning into single message
